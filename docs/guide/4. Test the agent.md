# Goal
Build confidence that your agent behaves correctly by covering it with unit, integration, and end-to-end tests.

## 1. Testing strategy overview
- **Unit tests** live under `agent/tests/unit/` and focus on isolated logic (`logic.py`, `decision.py`, tools with mocked I/O).
- **Integration tests** live under `agent/tests/integration/` and exercise multiple components together (runtime + chain of responsibility + mocked external systems).
- **End-to-end smoke tests** run the FastAPI app via `httpx.AsyncClient` or Docker Compose to ensure routes respond correctly.

> **Remember:** Start small. A failing unit test is faster to debug than a failing integration test.

## 2. Unit tests
1. **Logic functions** (`agent/src/custom/agent/logic.py`)
   ```python
   def test_score_backup_signals_returns_high_score_when_tags_present():
       asset = make_asset(backups=["s3"], tags=["backup-enabled"])
       score = score_backup_signals(asset)
       assert score.value >= 0.8
   ```
2. **Decision predicates** (`agent/src/custom/agent/decision.py`)
   - Mock `dedup_store` to simulate first-time vs. duplicate events.
   - Assert that thin and fat envelopes are handled correctly.
3. **Tools** (`agent/src/custom/agent/tools.py`)
   - Use `respx` or `pytest-httpx` to mock HTTP calls.
   - Verify error handling and retries.

Run all unit tests:
```bash
pytest agent/tests/unit -vv
```

## 3. Integration tests
Create `agent/tests/integration/test_event_flow.py` to cover thin and fat event processing end to end (with external services mocked):

```python
@pytest.mark.asyncio
async def test_thin_event_fetches_asset_and_publishes_result(publish_mock, httpx_mock):
    httpx_mock.get("https://gateway/assets/a-123", json=asset_payload)

    envelope = EventEnvelopeThin(...)
    result = await process_event(envelope)

    publish_mock.assert_called_once_with(result_topic, expected_result_payload)
    assert result.decision == Decision.REQUIRES_BACKUP
```

Key scenarios:
- Thin event happy path (gateway succeeds).
- Thin event gateway timeout with retry + DLQ fallback.
- Fat event direct processing.
- Duplicate event ignored via idempotency cache.

Run integration suite:
```bash
pytest agent/tests/integration -vv
```

## 4. End-to-end smoke tests
Use FastAPI’s `AsyncClient` to hit the HTTP interface without Docker:

```python
@pytest.mark.asyncio
async def test_check_backup_endpoint_returns_200(async_client):
    payload = AssetMetadata(...).model_dump()
    response = await async_client.post("/check-backup", json=payload)
    assert response.status_code == 200
```

For deployment pipelines, add a smoke test script that calls the live endpoint and asserts health checks (`/actuators/health`).

## 5. Coverage & quality gates
- Configure `pyproject.toml` to target ≥ 90% coverage on critical modules (`logic.py`, `decision.py`, `runtime.py`).
- Fail CI if linting or tests fail by integrating with `.pre-commit-config.yaml` and the Azure Pipeline.

## 6. Helpful commands
```bash
make lint            # Ruff + Black
make test            # Runs pytest suites defined in Makefile
pytest --maxfail=1   # Stop early on failure
pytest -k "thin"     # Focus on a subset when iterating
```
